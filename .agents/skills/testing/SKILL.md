# テスト規約

- テストフレームワーク: pytest
- 実行: `uv run pytest`

## 構造
- テストファイル名: `test_<module>.py`
- テスト関数名: `test_<動作の説明>`（例: `test_空リストで空を返す`）
- Arrange-Act-Assert（AAA）パターンに従う

## ベストプラクティス
- 実装の詳細ではなく振る舞いをテストする
- 外部依存はモック、内部モジュールはモックしない
- テストデータにはファクトリやフィクスチャを使う
- 副作用は適切にクリーンアップ
